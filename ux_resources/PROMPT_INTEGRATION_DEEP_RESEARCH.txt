# CONTEXTE

J'ai un projet de UX Rulebook avec des règles consolidées de design.
Tu vas m'aider à intégrer un nouveau fichier de règles UX obtenu via ChatGPT Deep Research.

## STRUCTURE ACTUELLE

```
c:\Users\ludov\.infernal_wheel\ux_resources\
  WEB.md           ← ~260 règles web (WCAG, patterns, accessibility)
  MOBILE.md        ← ~320 règles mobile (iOS HIG, Material 3)
  DESIGN_TREE.md   ← Arbres de décision ASCII (mind map)
  COMMANDES.txt    ← Doc utilisateur
  prompts\
    PROMPT_REPRISE.md  ← Prompt de reprise projet
```

## FORMAT DES FICHIERS EXISTANTS

Les fichiers utilisent ce format grep-able:

```markdown
### X. Titre Section

| Pattern | Règle | Valeur | Source |
|---------|-------|--------|--------|
| ... | ... | ... | ... |

**Checklist:**
- [ ] Item 1
- [ ] Item 2
```

Avec:
- Numérotation des sections (### 1., ### 2., etc.)
- Tables markdown pour les règles
- Valeurs CONCRÈTES (pixels, ms, ratios) - jamais de "approprié" ou "suffisant"
- Sources citées
- Checklists avec [ ]

## CE QUE TU DOIS FAIRE

1. **Lire le fichier de deep research** que je vais te donner

2. **Vérifier les doublons** - ne pas ajouter ce qui existe déjà dans WEB.md ou MOBILE.md
   - Grep dans les fichiers existants avant d'ajouter
   - Si doublon: garder la version la plus complète/récente

3. **Intégrer les nouvelles règles** selon le sujet:
   - Data Visualization → WEB.md (nouvelle section)
   - Microcopy → WEB.md (nouvelle section)
   - Gamification/Habits → MOBILE.md (nouvelle section)
   - Perceived Performance → WEB.md (ajouter à section existante ou nouvelle)
   - i18n/RTL → WEB.md + MOBILE.md (les deux)
   - Multi-device → WEB.md (nouvelle section)
   - AI/ML UX → WEB.md (nouvelle section)
   - Error Recovery/Undo → WEB.md (nouvelle section)
   - Cognitive Accessibility → WEB.md (ajouter après WCAG)
   - Advanced Motion → MOBILE.md (ajouter après section motion existante)

4. **Reformater si nécessaire** pour matcher le style existant:
   - Ajouter numéros de section qui suivent la numérotation actuelle
   - Convertir en tables markdown si pas déjà fait
   - S'assurer que les valeurs sont concrètes

5. **Mettre à jour DESIGN_TREE.md** si nouvelles décisions à ajouter (arbres ASCII)

6. **Mettre à jour PROMPT_REPRISE.md** si nouvelles sections importantes

## WORKFLOW
